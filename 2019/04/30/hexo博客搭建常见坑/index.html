hexo博客搭建常见坑

发表于 2019-04-30 | 更新于 2019-05-01

repo嵌套

hexo整个项目文件夹和public文件夹是两个repo，hexo d是自动上传public文件夹

显示目录

书写时需注意，元数据中的title是一级标题，正文中最高的标题为二级标题

经hexo-blog-encrypt加密后的文章，输入密码之后也无法显示TOC

在主题的_config文件里面，将toc的enable设置为true

蜜汁故障排除

删除.deploy_git文件夹并且执行hexo clean后，再重新执行hexo d

先hexo clean再hexo g，进入public，不用自动部署方式，使用git add .和git commit -m ""、git push origin master最原始的方法不会出错的。最后访问的博客其实是静态文件，所以要访问正常，你提交的其实是public文件下的静态文件

hexo原生方法插入图片

修改_config.yml配置文件post_asset_folder项为true

使用new命令新建文章，或者手动创建与已有博客同名的文件夹

next主题显示德文

根目录下的_config文件的language选项默认留空，手动指定即可，语言名在/themes/next/languages

每次部署都要配置域名

在source文件夹下新建文件，文件名为CNAME，无扩展名，文件内容为自定义域名

next主题代码高亮无法正常显示

反三点后面加语言名

部署时提示deloyer not found: git

npm i hexo-deployer-git –S

参见官方文档

yml冒号后面要有空格

ouc-web-dev需求
hexo引入live2d挂件
