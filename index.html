hexo引入live2d挂件

发表于 2019-05-01 | 更新于 2019-05-02

安装

环境：npm i -S hexo-helper-live2d

现有模型：npm i -S live2d-widget-model-shizuku

注意，无需安装或引入live2d-widget.js，虽然用到其中很多API

按照hexo-helper-live2d的readme即可顺利挂载一个npm上的现有模型

注意，yaml不允许有重复的键，所以要把文档中Helper-specific和General Settings中的配置据需混合

引入自定义模型（以Steam为例）

Steam的Face Rig创意工坊下载的内容位于Steam/steamapps/workshop/content/274920

直接复制到/live2d_models/模型名目录下即可

检查一下以model.json结尾的文件，其中的name字段要和全局配置文件中model.use的值一致

调试模型位置

按照live2d-widget.js的API调试模型位置

以下为官方demo的部分数值，比较合理（与默认值不同，默认值太小了）

dev.border: true用于显示画布边框，便于调试

dialog.enable和dialog.hitokoto用于对话框

频繁调试的方法

每次改完yaml中的参数后，都需要重新hexo g

PowerShell可以使用|分隔命令，一次执行多条

所以使用hexo g|hexo serve即可频繁调试

自定义模型存在的问题

重心偏高

外部模型有些是半身像，往往会导致重心偏高，挡住对话框

使用live2d.display.vOffset是解决不了的，因为对话框会跟着人物一起移动

根据hexo-helper-live2d的readme，曾经有live2d.model.hHeadPos和vHeadPos，但API文档中已经查不到，而且亲测无效，所以重心问题可能需要使用live2d自身技术解决

可以使用live2d.model.scale和live2d.mobile.scale进行折中，这两个参数只缩放人物模型

但是，如果缩小模型的同时增大画布，对话框也会随画布缩放，导致人物依然挡住对话框

所以，只能以缩小模型为代价，让对话框露出

以美穗为例，使用此折中方案时，两个scale均为0.75，vOffset为-125

加载缓慢

在线访问时，往往先会有对话框和文字出现，而模型需要数秒才能加载出来

无法换表情

npm上的模型，通过鼠标点击可以更换表情，但自定义模型没有此功能

hexo博客搭建常见坑

发表于 2019-04-30 | 更新于 2019-05-01

repo嵌套

hexo整个项目文件夹和public文件夹是两个repo，hexo d是自动上传public文件夹

显示目录

书写时需注意，元数据中的title是一级标题，正文中最高的标题为二级标题

经hexo-blog-encrypt加密后的文章，输入密码之后也无法显示TOC

在主题的_config文件里面，将toc的enable设置为true

蜜汁故障排除

删除.deploy_git文件夹并且执行hexo clean后，再重新执行hexo d

先hexo clean再hexo g，进入public，不用自动部署方式，使用git add .和git commit -m ""、git push origin master最原始的方法不会出错的。最后访问的博客其实是静态文件，所以要访问正常，你提交的其实是public文件下的静态文件

hexo原生方法插入图片

修改_config.yml配置文件post_asset_folder项为true

使用new命令新建文章，或者手动创建与已有博客同名的文件夹

next主题显示德文

根目录下的_config文件的language选项默认留空，手动指定即可，语言名在/themes/next/languages

每次部署都要配置域名

在source文件夹下新建文件，文件名为CNAME，无扩展名，文件内容为自定义域名

next主题代码高亮无法正常显示

反三点后面加语言名

部署时提示deloyer not found: git

npm i hexo-deployer-git –S

参见官方文档

yml冒号后面要有空格

系统概述

分为网页端和小程序端

网页端主要由管理员和助教使用，小程序端主要由学生和游客使用，管理员和助教也可以在小程序端实现简单管理

网页端

  1. 学生信息管理
  2. 导入学生名单
  3. 在线编辑并上传测试题
  4. 上传课件、视频等文件

用户权限

访问课程介绍 Y Y Y Y Y Y

发送通知

通过微信的应用消息通知推送给选课学生，包括下节课的安排和在线测试提醒等

上传文件

课件分享

可以在线浏览pdf/ppt，或者跳转到百度网盘

视频课程

一系列短视频，比如HTML5的科普动画，可在线观看

在线测试

由管理员或助教在网页版编辑并上传，设置参加测试的起止时间，通过应用消息通知推送给选课学生

题型为单选题或多选题，机器判断对错并给出分数

  3. 选课学期
  4. 缩略图
  5. 功能说明
  6. 所获奖项（如有）
